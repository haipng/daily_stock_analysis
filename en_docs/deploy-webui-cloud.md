# Cloud Server Web Interface Access Guide

If you've deployed the project to a cloud server but don't know what URL to enter in your browser to open the Web management interface, this tutorial is for you.

> Actually just two steps: make the service listen to external network, then enter the URL in browser.

## Method 1: Direct Deployment (pip + python)

### Step 1: Modify .env Listening Address

Open `.env` with editor (in project root directory, same as `main.py`), find this line:

```env
WEBUI_HOST=127.0.0.1
```

Change `127.0.0.1` to `0.0.0.0`:

```env
WEBUI_HOST=0.0.0.0
```

> `127.0.0.1` means only local machine can access; `0.0.0.0` means any source can access. Cloud servers must change to `0.0.0.0` to open interface from external network.

> **Note**: Current `python main.py` startup logic when host is default `0.0.0.0` reads `WEBUI_HOST` from `.env`; even with explicit `--host 0.0.0.0`, if `.env` still has `WEBUI_HOST=127.0.0.1`, final result may only listen locally. Cloud servers must change `.env` to `WEBUI_HOST=0.0.0.0` first.

### Step 2: Start Service

Execute in project root:

```bash
# Only start Web interface (no auto analysis)
python main.py --webui-only

# Or: Start Web interface (run analysis once at startup; add --schedule or set SCHEDULE_ENABLED=true for daily scheduled analysis)
python main.py --webui
```

After successful start, terminal displays similar:

```
FastAPI service started: http://0.0.0.0:8000
```

To keep service running after exiting terminal, use `nohup`:

```bash
nohup python main.py --webui-only > /dev/null 2>&1 &
```

> Log files automatically written to `logs/` directory; view with `tail -f logs/stock_analysis_*.log`.

### Change Port (Optional)

Default port is 8000. To use different port, set in `.env`:

```env
WEBUI_PORT=8888
```

Then restart service.

## Method 2: Docker Compose

### Step 1: Confirm .env Configuration Exists

Project's `docker/docker-compose.yml` already auto-sets `WEBUI_HOST=0.0.0.0` inside container; you don't need to change listening address in `.env`; Docker handles it automatically.

### Step 2: Start Service

Execute in project root:

```bash
# Start both scheduled analysis + Web interface (recommended)
docker-compose -f ./docker/docker-compose.yml up -d

# Or only start Web interface service
docker-compose -f ./docker/docker-compose.yml up -d server
```

Check status after starting:

```bash
docker-compose -f ./docker/docker-compose.yml ps
```

If `server` service status is `running`, Web interface is running.

### Change Port (Optional)

Default port is 8000. To use different port, set in `.env`:

```env
API_PORT=8888
```

Then restart container:

```bash
docker-compose -f ./docker/docker-compose.yml down
docker-compose -f ./docker/docker-compose.yml up -d
```

## How to Open Interface in Browser

After service starts, enter in browser address bar:

```
http://your_server_public_ip:8000
```

For example, if your server IP is `1.2.3.4`, enter:

```
http://1.2.3.4:8000
```

If your domain is already resolved to this server, can also directly use domain:

```
http://your-domain.com:8000
```

> **Where to find public IP?** Log into your cloud server control panel (Alibaba Cloud/Tencent Cloud/AWS, etc.); in instance list you can see "Public IP" or "Elastic IP".

## How to Confirm Docker Rebuild Took Effect

First distinguish two things:

1. **Docker image release version**: Check image tag you used during deployment, e.g., `ghcr.io/zhulinsen/daily_stock_analysis:v3.12.0`. Repository's Docker releases are triggered by `.github/workflows/docker-publish.yml` on `v*.*.*` Git tags, so Docker version should follow image tag / GitHub Releases.
2. **Currently-loaded frontend build**: Check version info card in WebUI "System Settings" page to confirm if browser got updated static resources.

That is, **"System Settings" version info is better for judging frontend rebuild success, not equivalent to Docker image release version**.

WebUI now displays read-only "Version Info" card in "System Settings" page, containing:

- `WebUI Version`
- `Build ID`
- `Build Time`

If version number in `apps/dsa-web/package.json` is still placeholder `0.0.0`, page auto-fallback displays `Build ID` generated by this frontend build, avoiding misreading placeholder as actual release version.

When you re-execute `docker-compose -f ./docker/docker-compose.yml up -d --build`, or re-execute frontend `npm run build` separately, can refresh browser and enter "System Settings"; first confirm if "Build Time" has changed; if yes, usually means current-loaded static resources switched to latest build.

To confirm "which official version am I actually running", prioritize these methods:

```yaml
# Method 1: Check image tag in docker-compose / deployment script
image: ghcr.io/zhulinsen/daily_stock_analysis:v3.12.0
```

```bash
# Method 2: Check your pull command
docker pull ghcr.io/zhulinsen/daily_stock_analysis:v3.12.0
```

If you always use `latest`, recommend switching to explicit version tag; otherwise very hard to tell if you already re-updated to same version based only on page info.

When confirming local frontend packaging chain, recommend executing:

```bash
cd apps/dsa-web
npm ci
npm run lint
npm run build
```

Where `build` output generates `index.html`/JS/CSS resources under `static` containing build time and version info; refresh should show change in "Version Info" card.

## Can't Access? Check These

### 1. Security Group / Firewall Didn't Allow Port

This is the most common cause. Cloud servers default only open port 22 (SSH); need to manually allow port 8000 (or your custom port).

**Operation** (Alibaba Cloud example):
1. Login Alibaba Cloud console → ECS → Find your instance
2. Click "Security Group" → "Configure Rules" → "Add Security Group Rule"
3. Direction select "Inbound", port range fill `8000/8000`, Authorization object fill `0.0.0.0/0`, click "OK"

Tencent Cloud, AWS, etc. are similar; find "Security Group" or "Firewall Rules", add inbound rule allowing TCP port 8000.

### 2. Server System Firewall Blocked

If your system has `ufw` or `firewalld` enabled, also need to allow port:

```bash
# Ubuntu / Debian (ufw)
sudo ufw allow 8000

# CentOS / RHEL (firewalld)
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

### 3. .env WEBUI_HOST Not Changed in Direct Deploy

Second most common cause. `.env` defaults to `WEBUI_HOST=127.0.0.1`, so service only listens locally; external network can't connect.

Fix: Open `.env`, change `WEBUI_HOST=127.0.0.1` to `WEBUI_HOST=0.0.0.0`, restart service.

> Docker method doesn't need this; skip.

### 4. Port Numbers Don't Match

Check access URL port matches `.env` / startup command configured port.

- Direct deploy: default 8000, change with `WEBUI_PORT=xxxx`
- Docker: default 8000, change with `API_PORT=xxxx`

### 5. Page Opens but UI Elements Abnormally Large / Layout Broken

**Symptom**: Browser can access port 8000, page has content, but text, buttons, cards abnormally large, no normal layout and styling.

**Root Cause**: `static/index.html` exists but CSS/JS resources missing (`static/assets/` empty or not exist); browser loaded HTML framework but can't get styles and scripts, degrading to bare HTML rendering.

Can first use browser dev tools (F12 → Network tab) to check if there are `/assets/index-*.js`, `/assets/index-*.css` with **404** errors. If yes, fix per these methods:

**Docker users**:

```bash
docker-compose -f ./docker/docker-compose.yml down
docker-compose -f ./docker/docker-compose.yml build --no-cache
docker-compose -f ./docker/docker-compose.yml up -d
```

After rebuild, use `Ctrl+Shift+R` to force refresh browser cache, then visit page.

**Direct deploy users**: First ensure Node.js 18+ installed (recommend 20+), then manually build frontend:

```bash
cd apps/dsa-web
npm ci
npm run build
cd ../..
python main.py --webui-only
```

## Optional: Nginx Reverse Proxy (Bind Domain / Port 80)

If you have domain or don't want `:8000` in URL, can use Nginx reverse proxy; forward port 80/443 traffic to backend service.

### Install Nginx

```bash
# Ubuntu / Debian
sudo apt update && sudo apt install -y nginx

# CentOS
sudo yum install -y nginx
```

### Configuration File Example

Create file `/etc/nginx/conf.d/stock-analyzer.conf`, content (replace `your-domain.com` with your domain or IP):

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Support WebSocket (Agent chat page needs this)
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```
